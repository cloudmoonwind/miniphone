#### 梦境星星（未解读）

夜空中漂浮着**各种样式的星星**，每颗星星代表一个未解读的梦境：

**十字星 ✦**：

**五角星 ★**：

**六角星 ✦**（雪花状）：

#### 星星的运动

每颗星星都在缓慢**随机飘动**：
- 移动轨迹是平滑的贝塞尔曲线
- 速度很慢（10-20秒移动几个像素）
- 偶尔会稍微闪烁（透明度变化）
- 星星之间不会重叠（有轻微的排斥力）
- 移动范围限定在夜空区域内

**视觉效果**：
- 每颗星星有柔和的外发光（glow效果）
- 光晕大小会轻微呼吸（scale微微变化）
- 鼠标移到星星附近，星星会稍微变亮（hover效果）

### 点击星星的完整流程

#### 第一步：星星被选中

点击某颗星星时：
- 星星立刻停止飘动
- 开始**旋转**（顺时针，缓慢优雅）
- 同时**变亮**（光晕增强）
- 其他星星稍微变暗（突出这颗）

#### 第二步：星星飞向屏幕

星星保持旋转，开始**飞向屏幕中心**：
- 移动轨迹是流畅的曲线（ease-in-out）
- 飞行过程中逐渐**放大**
- 光芒拖出淡淡的轨迹（粒子效果）
- 飞行时间约1秒

#### 第三步：展开为悬浮框

星星到达屏幕中心后：
- 旋转加速，光芒爆发（小范围的光效）
- 然后**展开变形**成一个大的圆角矩形悬浮框（变形动画流畅）
- 悬浮框背景是**半透明毛玻璃**（backdrop-filter: blur）
- 边缘有星星原本的颜色作为细细的发光描边
- 悬浮框大小约占屏幕的70-80%

**主体**：

#### 第五步：悬浮框收起

function drawStar(cx, cy, size, time){
const img = ctx.createImageData(size, size);
const data = img.data;

for(let i=0;i<size;i++){
for(let j=0;j<size;j++){
let x = (i - size/2) / (size/2);
let y = (j - size/2) / (size/2);

  // ===== 内凹菱形核心 =====
  let diamond = Math.pow(Math.abs(x), 0.7) + Math.pow(Math.abs(y), 0.7);
  let core = Math.exp(-diamond * 6);

  // ===== 十字光束 =====
  let beamX = Math.exp(-Math.abs(y)*25) / (1 + Math.abs(x)*3);
  let beamY = Math.exp(-Math.abs(x)*25) / (1 + Math.abs(y)*3);
  let cross = beamX + beamY;

  // ===== 外层光晕（内凹菱形）=====
  let haloShape = Math.pow(Math.abs(x), 0.6) + Math.pow(Math.abs(y), 0.6);
  let halo = Math.exp(-haloShape * 2);

  // ===== 闪烁 =====
  let flicker = 1 + 0.05 * Math.sin(time*3);

  let b = (core*1.5 + cross*0.8 + halo*0.6) * flicker;

  // ===== 颜色（白 → 青 → 蓝）=====
  let r = b * 255;
  let g = b * 220;
  let bl = b * 255 + halo*100;

  let idx = (j*size + i)*4;
  data[idx] = Math.min(255, r);
  data[idx+1] = Math.min(255, g);
  data[idx+2] = Math.min(255, bl);
  data[idx+3] = Math.min(255, b*255);
}

}

ctx.putImageData(img, cx - size/2, cy - size/2);
}

## 关于大富翁的所有问题，逐条回答

### 一、两套AI系统是怎么设计的？（line 282，line 297）

**主持人AI（callHostAI）和角色AI（callCharAI）是完全分开的两个函数，各自独立上下文和历史记录。**

**主持人AI的上下文：**
- system：模式风格说明（益智/恋爱/十八禁各有不同语气定义）、游戏情境（线上/线下）、当前格子的场景文字（wbContent）、叙事要求
- history：game.hostHistory，最近10条主持人输出记录
- user：当前事件描述（比如"玩家掷出5，移动到咖啡厅，触发property事件"）
- 主持人只看游戏事件和自己的叙事历史，不看角色人设

**角色AI的上下文：**
- system：角色人设（description/persona）+ 世界书激活条目（正常的关键词/始终激活走一遍）+ 最近生活日志 + 游戏背景（当前回合、双方分数和位置）
- history：角色真实聊天记录（最近10条）+ game.charHistory（最近16条合并）
- user：用户在游戏中发的消息
- 角色AI才接入世界书，主持人AI不需要

**为什么这样设计：** 主持人是旁白/裁判，不是角色，不应该知道角色的"真实内心"。角色是有人设的活人，需要连续性和记忆。两套分开，互不干扰，各自的temperature也可以不同（主持人0.92更有创意，角色0.88更稳定）。

---

### 二、预设怎么用？（line 282，line 295，line 3）

**大富翁不使用提示词预设系统（SettingsApp里的那套）。** 提示词预设是用于普通聊天的——把系统提示词、角色人设、上下文结构存在一起复用。大富翁的提示词是在代码里动态拼接的（因为每次都包含当前格子/当前分数/当前局面，是动态的，不适合做静态预设）。

大富翁只需要**API预设**（model名字/API key/base URL这些连接参数）。在`getPreset(id)`里：
- 如果游戏创建时没有指定`hostPresetId`/`charPresetId`，自动fallback到`active.primaryPresetId`
- 所以只要有一个配置好的主预设，大富翁就能运行，不需要单独建"大富翁专用预设"

**如果你想为大富翁配不同的模型**（比如主持人用一个模型，角色用另一个），在游戏创建界面可以选。默认都走同一个主预设。

---

### 三、世界书和棋盘格子怎么联动？（line 295）

这是你纠结了很久的问题，直接说结论：

**当前设计：游戏创建时把世界书内容"烘焙"进每个格子（embedWbContent），游戏过程中只把当前格子的文字发给AI，不发全棋盘。**

具体流程：
1. 创建游戏时：`embedWbContent(cells, wbBookId, mode)` 运行一次
2. 它读取选中的世界书的所有条目，按**格子名称**匹配（比如格子名"咖啡厅"对应同名条目），或按关键词匹配（条目关键词包含"格子4"或"4"）
3. 每个格子的 `cell.wbContent` 字段被填充为对应条目的内容
4. 如果没有对应条目，fallback到内置的 `WB_CONTENT[mode][i]`（三种模式各25条）
5. 游戏中：主持人AI只收到 `cell.wbContent`（当前格子的那一条），不是全部25条

**为什么不把全棋盘发给AI？** 会严重浪费token并分散注意力。主持人只需要知道"我现在在咖啡厅，要发生什么"，不需要知道20格之后有什么。

**世界书其他条目（非格子内容）能用吗？** 能。世界书里除了格子条目，还可以添加其他条目（比如"世界观设定""主持人风格规则"），设为始终激活或关键词激活，这些会走正常的worldbook激活逻辑注入到**角色AI**的上下文。主持人AI不走worldbook激活，它的上下文只有格子文字和模式风格定义。

**世界书条目怎么对应格子：** 一条目一格子，条目名称 = 格子名称（比如"咖啡厅"，"书店"，"十字路口"），内容写这个格子的叙事文字。或者条目关键词写"格子4"这样精确对应。

**全棋盘写一个条目行不行？** 技术上能匹配，但主持人每次都会收到全部25条格子的文字（因为条目一旦激活就全部注入），上下文很长，AI很可能只看第一段然后忽略后面的。不推荐。

### 四、功能做了什么、没做什么（对应小克大富翁规格，line 303-834）

### 五、世界书功能完善（line 836）

世界书的级联激活逻辑**已经按你描述的实现了**（在 `server/services/worldbook.js`）：

**已实现的逻辑：**
- `scanDepth`：每本书有扫描深度设置（默认20），只扫最近n条消息
- `noRecurse`：此条目不被其他条目的内容激活（创建时可设置）
- `noFurtherRecurse`：此条目的内容不会去激活其他条目（创建时可设置）
- 多轮激活循环：第一轮 → 收集始终激活条目 → 用所有激活条目的内容扫描关键词 → 再次激活匹配的条目 → 循环直到没有新条目被激活

**前端世界书应用（FilesApp）缺少这些设置的UI**：目前创建/编辑条目弹窗里没有`noRecurse`/`noFurtherRecurse`的勾选框，也没有书级别的`scanDepth`输入框。这是下一步要做的。

### 六、为什么AI叙事一直不触发（currentNarrative: null）

**根本原因：代码改了但服务器没重启。** 所有修复（包括`getPreset`的fallback逻辑）都写在代码里了，但你测试时跑的是改之前的服务器进程，旧代码里`getPreset(null)`直接返回null，AI调用从未真正执行过。

**需要你做的事：重启Node服务器（Ctrl+C然后重新npm run dev/start）**，然后新建一局游戏，掷骰子，主持人叙事就会出现。

### 七、世界书app里为什么没有大富翁世界书

在DafuApp第一次打开时会自动调用`/api/dafu/init-worldbooks`创建三个世界书（大富翁·华尔街征途/约会之旅/欲望迷宫），每个各含25条格子内容。但如果你先打开世界书app、没有打开DafuApp，世界书app里就是空的。修复方法：先打开一次DafuApp，或者服务器启动时自动初始化。
