## 二、知识库视觉风格（纸墨书卷主题）

### 底图与装饰

- 使用 `素材/纸纹底素_淡黄.jpg`（已复制到 `client/public/paper-bg.jpg`）作为全局背景
- **左侧书脊**（`SpineStrip`）：三层渐变，最外缘深棕线 `rgba(22,8,2,0.6)` + 装订阴影渐变 + 内侧缝合细线
- **右侧页边**（`PageStack`）：单条深线 + 渐变阴影，营造书页厚度感但不"凸出"
- **内阴影**（`InnerShadow`）：`linear-gradient` 左深右淡，模拟翻开书页的弯曲弧面

### 书名标题区（BookDetail）

- 无顶部导航栏，沉浸式；书名 `《书名》` 居中，点击即返回书列表
- 标题区有与书脊同色调的深色背景渐变 `rgba(38,18,6,0.10→0)`
- 双分隔线区分标题区与条目区：主线 `rgba(60,32,12,0.28)` + 副线 `rgba(60,32,12,0.10)`

### 条目列表

- 条目间分隔改为**暖棕色虚线 + 中心小菱形**（`EntryDivider`），在纸纹上清晰可见
- 条目之间无圆角、无边框卡片

## 三、条目卡（EntryCard）交互

### 左滑操作

- **触控**：`onTouchStart/Move/End` 原生触摸事件，向左滑超过 SNAP×50% 则吸附展开
- **鼠标**：`onMouseDown` + document `mousemove/mouseup` 监听器，拖动超过 6px 识别为拖拽
- 操作区（启用/复制/删除）用 `opacity + pointerEvents` 渐显，不用 `overflow-hidden`（避免裁剪下拉框）

### 下拉框穿透修复

- **根本原因**：`translateX(0)` 在父元素上创建 stacking context，导致 `position:fixed` 子元素定位失效
- **解决方案**：条目主体 div 仅在 `slideX !== 0` 时才设置 transform；`InlineDropdown` 改用 `position:fixed` + `getBoundingClientRect()` 计算弹出位置

### 策略圆点

- 从行1右侧移至**左侧书脊旁**（`absolute left:17px top:4`）
- 绿色实心 = 关键词触发，棕色空心 = 常驻，点击调 API 切换并刷新

### 行内元数据

- 行1：条目名称（`font-semibold`）+ 开书图标（`BookSVG`），左padding=28px（`pl-7`）避开圆点
- 行2：插入位置下拉 + 排序数 + 概率，斜体淡色，行内可编辑，`pl-7 pr-4`

### 关键词区布局

- 第一行：`关键词：` 标签 + 逗号分隔输入框
- 第二行：过滤逻辑下拉（任一/全部/排除…）+ 过滤词输入框，**左对齐**，与第一行标签对齐，无缩进
- 去掉了"条件："文字标签，保持简洁

