# 数值系统设计想法

**日期**: 2024-03-26  
**来源**: Agnes的夜晚灵感

---

## 核心设计思路

数值系统的每个数值包含以下内容：

1. **名称**（给用户看、可修改的）
2. **变量名**（代码里写的）
3. **描述**（这个数值是什么意思）
4. **区间范围**（比如[-100, 100]或[0, 100]）
5. **分段描述**（不同数值区间的解释说明）
6. **更新规则**（什么情况下加减多少）

---

## 示例：好感度

**名称**: 好感度  
**变量名**: affection  
**描述**: 角色对用户喜欢/关爱或厌恶/憎恨的感情  
**区间**: [-100, 100]

### 分段描述

不同阶段的描述（提示词）只有在数值到达该阶段时才会触发：

1. [-100, -80]: 难以忍受对方相关的任何内容，......
2. ......（其他阶段待补充）

### 更新规则

- xxx加1-2
- xxx加5-10
- xxx减1-20
- 等等

备注：可以做成一套整体的提示词，也可以做成根据数值区间选择性发送的。

### 心情/情绪相关

心情这一块最复杂，因为人总是五味杂陈，很多情绪可以同时出现。

设计思路：
- 要尽可能选择对立的两端
- 有些情绪组合会十分美味（比如"平静的愤怒"）
- 开心/伤心这种看似对立的会很难搞：
  - 0可以是既开心又伤心
  - 也可以是没什么情绪
  - 所以用在数值系统的并不是纯粹的情绪词

情绪词不需要100分的数值，打算更主要做成选择，或者说**饼图**。

（想到了小说里经典的"四分不屑三分凉薄三分讥讽"）

### 已想到的情绪对立对

- **理智/失智** - 想起来一些克苏鲁小说里面的SAN值

- **镇定/激动**

稳定和镇定有点区别：
- 比如"稳定的激动"想要形容的是一个高强度持续输出的感觉
