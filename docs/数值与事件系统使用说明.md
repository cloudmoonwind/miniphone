# ICS 数值与事件系统 — 使用说明

本文档面向两类用户：
- **作者**（角色创作者）：如何设计数值、事件、规则
- **玩家**（角色使用者）：系统会怎样影响你的对话体验

---

# 给作者的说明

## 一、数值系统

### 什么是数值？

数值是角色身上可以量化的状态。比如好感度 0-100，精力 0-100。数值会影响角色的行为方式和对话态度。

### 创建数值

在道枢 App 中，点击「新建」可以为角色创建一个数值。需要填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 变量名 | 英文标识，用于内部引用，不可重复 | `affection` |
| 显示名 | 玩家看到的名字 | 好感度 |
| 分类 | 数值属于哪一类 | 关系类 |
| 范围 | 最小值 ~ 最大值 | 0 ~ 100 |
| 初始值 | 角色刚创建时的默认值 | 50 |

**分类说明：**

| 分类 | 英文 | 用途 |
|------|------|------|
| 内在属性 | attribute | 变化慢的固有特质（智力、社交能力） |
| 动态状态 | status | 随时变化的状态（体力、饥饿） |
| 心理状态 | emotion | 情绪和心理（心情、压力、理智） |
| 关系类 | relation | 角色对特定对象的态度（好感度） |
| 社会属性 | social | 社会维度的属性（声誉、阅历） |

### 数值阶段

每个数值可以划分阶段。不同阶段有不同的描述和提示词，AI 会据此调整角色的行为。

**好感度的阶段示例：**

| 范围 | 阶段名 | 描述 | 提示词片段 |
|------|--------|------|------------|
| 0-20 | 陌生 | 对你保持距离 | 对用户态度冷淡，保持社交距离 |
| 20-50 | 认识 | 愿意交谈 | 对用户态度礼貌，愿意简单交流 |
| 50-80 | 朋友 | 态度温和 | 对用户态度温和，愿意分享日常 |
| 80-100 | 亲密 | 非常热情 | 对用户非常热情，会主动关心 |

**要点：**
- 阶段范围应该覆盖数值的整个区间，不要留空隙
- 提示词片段会被自动注入到 AI 的上下文中，直接影响角色表现
- 阶段描述是给你自己看的备注，不会发给 AI

### 在提示词中引用数值

在角色提示词或世界书中，你可以用占位符引用数值：

```
{{v:affection}}        → 替换为当前数值（如 75）
{{v:affection:stage}}  → 替换为阶段名（如"朋友"）
{{v:affection:desc}}   → 替换为阶段描述（如"态度温和"）
{{v:affection:prompt}} → 替换为提示词片段
```

**使用示例：**

在角色设定中写：
```
{{char}}对{{user}}的好感度为{{v:affection}}，目前处于「{{v:affection:stage}}」阶段。
{{v:affection:prompt}}
```

系统会自动替换成：
```
小明对你的好感度为75，目前处于「朋友」阶段。
对用户态度温和，愿意分享日常
```

### 变化规则

规则控制数值在什么情况下自动变化。每条规则包含：

| 字段 | 说明 |
|------|------|
| 触发时机 | 什么时候检查这条规则 |
| 生效范围 | 数值在什么范围内这条规则才生效（留空=全范围） |
| 操作 | `add`（加减）、`set`（设为固定值）、`multiply`（乘以） |
| 数量 | 操作的数值 |

**触发时机：**

| 时机 | 说明 |
|------|------|
| `chat_end` | 每轮对话结束后 |
| `time_pass` | 时间流逝（每小时/每天） |
| `receive_gift` | 收到礼物时 |
| `event_complete` | 事件完成时 |
| `value_change` | 其他数值变化时 |

**示例 — 好感度的渐变规则：**

| 范围 | 触发 | 操作 | 说明 |
|------|------|------|------|
| 0-20 | chat_end | add +3 | 陌生期，聊天涨好感快 |
| 20-50 | chat_end | add +2 | 认识期，涨幅中等 |
| 50-80 | chat_end | add +1 | 朋友期，涨幅变小 |
| 80-100 | chat_end | add +0.5 | 亲密期，很难再涨 |
| 全范围 | time_pass | add -0.5 | 长时间不聊天，好感缓慢下降 |

## 二、事件系统

### 什么是事件？

事件是你预先设计的剧情节点。它们不会立刻发生，而是在特定条件满足时被触发，改变角色的行为和状态。

### 事件的生命周期

```
锁定 (locked)
  ↓  满足解锁条件
待触发 (pending)
  ↓  满足触发条件 + 概率判定通过
进行中 (active)     ← 多步骤事件才会经过此状态
  ↓  所有步骤完成
已完成 (completed)
  ↓  如果可重复且冷却结束
回到 待触发
```

### 创建事件

每个事件需要一个唯一 ID（如 `evt_confession`）和基本信息：

| 字段 | 说明 |
|------|------|
| ID | 唯一标识，建议用 `evt_` 前缀 |
| 名称 | 事件名，如"表白时机" |
| 描述 | 你的备注 |
| 优先级 | 数字越大越优先触发（多个事件同时满足时） |
| 概率 | 0-100，条件满足后触发的概率 |
| 可重复 | 是否允许多次触发 |
| 最大触发次数 | 留空=无限 |

**概率的用法：**
- 100 = 条件满足必定触发（用于关键剧情）
- 30 = 条件满足后 30% 概率触发（用于随机事件）

### 解锁条件 vs 触发条件

事件有两层条件，分开设计是为了更精细地控制时机：

- **解锁条件**：满足后事件从「锁定」变为「待触发」。通常是前置事件完成等静态条件
- **触发条件**：解锁后，还需要满足这些条件才真正触发。通常是数值、时间等动态条件

**示例 — "表白时机"事件：**

解锁条件：
- "第一次约会"事件已完成

触发条件：
- 好感度 >= 80
- 时间 = 晚上

意思是：先得完成约会（解锁），然后等好感度够高、在晚上的时候才会触发表白。

### 条件格式

条件支持「且/或」组合：

```
（好感度 >= 80 且 晚上）或（"喝醉事件"已完成）
```

对应的 JSON：
```json
{
  "组间逻辑": "或",
  "条件组": [
    {
      "组内逻辑": "且",
      "条件": [
        { "类型": "数值", "目标": "affection", "比较": ">=", "值": 80 },
        { "类型": "时间", "值": "evening" }
      ]
    },
    {
      "条件": [
        { "类型": "事件", "目标": "evt_drunk", "状态": "completed" }
      ]
    }
  ]
}
```

**条件类型：**

| 类型 | 检查什么 | 示例 |
|------|---------|------|
| 数值 | 角色数值 | 好感度 >= 80 |
| 标记 | 布尔标记 | confessed = true |
| 事件 | 事件状态 | evt_date 已完成 |
| 时间 | 当前时段 | morning / evening |
| 日期 | 当前日期类型 | weekday / weekend |
| 天气 | 当前天气 | rain / sunny |
| 地点 | 角色位置 | home / school |
| 关键词 | 对话中出现的词 | "喜欢" |

### 触发效果

事件触发后可以产生多种效果：

| 效果类型 | 说明 | 示例 |
|---------|------|------|
| 注入 | 向 AI 上下文插入文本 | "角色看着你，似乎有话想说..." |
| 改数值 | 修改角色数值 | 好感度 +10 |
| 设标记 | 设置布尔标记 | confessed = true |
| 触发事件 | 立刻触发另一个事件 | 触发"确定关系" |
| 解锁事件 | 解锁另一个事件 | 解锁"婚礼" |
| 锁定事件 | 锁回一个事件 | 锁定"分手" |

### 注入（Injection）

注入是事件最重要的效果之一。它会在 AI 生成回复前，把一段文本插入到提示词中，引导角色做出特定行为。

**注入位置：**

| 位置 | 说明 |
|------|------|
| before_char | 角色设定之前 |
| after_char | 角色设定之后 |
| status_section | 状态区域 |
| before_history | 历史消息之前 |
| depth | 插入到历史消息的指定深度 |

**持续类型：**

| 类型 | 说明 |
|------|------|
| once | 用一次就消失 |
| turns | 持续 N 轮对话 |
| until_event | 直到某个事件发生才消失 |
| permanent | 永久 |

### 事件连接

事件之间可以建立连接，形成剧情网络：

| 连接类型 | 说明 | 图中表示 |
|---------|------|---------|
| next | A 完成后 B 是下一个 | 实线箭头 |
| branch | A 可以走向 B（分支） | 虚线箭头 |
| trigger | A 完成时自动触发 B | 粗线箭头 |
| unlock | A 完成后 B 解锁 | 实线箭头 |

### 事件标签

用标签给事件分类，方便筛选和管理：

| 标签类型 | 说明 | 示例 |
|---------|------|------|
| line | 剧情线 | 主线1、支线A |
| category | 分类 | 情感、日常、战斗 |
| chapter | 章节 | 第一章 |

### 冷却机制

可重复事件需要冷却，防止反复触发：

- **按时间**：触发后 N 天才能再触发（如"每周一次的约会"）
- **按轮次**：触发后聊 N 轮才能再触发
- **条件冷却**：条件满足 N 次后才真正触发（如"下雨 3 次才偶遇"）

按时间/轮次二选一，条件冷却可以叠加。两个冷却都清零才能再次触发。

## 三、世界状态

世界状态是全局的键值对，不属于某个角色，但会影响所有角色。

常用的世界状态：

| 键 | 值示例 | 说明 |
|----|--------|------|
| weather | rain | 当前天气 |
| location | school | 当前地点 |
| time_period | evening | 当前时段 |
| season | summer | 当前季节 |

世界状态可以作为事件的触发条件（如"天气=雨天"时触发偶遇事件）。

## 四、设计技巧

### 1. 从简单开始

第一次使用建议只创建 2-3 个核心数值（好感度、心情、精力），配上基础阶段和简单规则。确认跑通后再慢慢加。

### 2. 阶段提示词要具体

不要写"态度好"，要写"对用户说话时会带笑意，偶尔开玩笑，愿意分享今天发生的事"。越具体，AI 表现越准确。

### 3. 事件概率不要太低

概率设太低（如 5%）会导致玩家很难遇到这个事件。建议随机事件至少 20-30%。

### 4. 用事件链构建故事线

```
初遇 → 第一次约会 → 表白
                   ↘ 误会（分支）→ 和好 → 表白
```

主线事件用 `next` 连接，分支用 `branch`。分支最终汇入主线即可。

### 5. 善用注入的持续类型

- 重要剧情转折：用 `permanent`（确定关系后永久改变态度）
- 短期情绪波动：用 `turns`（吵架后 3 轮内态度冷淡）
- 一次性提醒：用 `once`（通知角色某件事发生了）

# 给玩家的说明

## 数值系统

角色有多项数值反映当前状态，比如好感度、精力、心情等。这些数值会随着你和角色的互动自然变化：

- **聊天**会影响好感度和心情
- **长时间不互动**可能导致好感度缓慢下降
- **特殊事件**（送礼物、完成任务）会带来较大的数值变化

你可以在「道枢」App 中查看角色当前的所有数值。数值变化会直接影响角色对你的态度和行为——好感度低时角色会冷淡，高时会热情。

## 事件系统

在你和角色的对话过程中，有时会触发特殊事件。事件是作者预先设计的剧情节点，在条件满足时自动发生。

**你可能会遇到的情况：**

- 角色突然提起某个话题或表现出异常行为 → 这可能是一个事件被触发了
- 角色的态度突然发生变化 → 可能是数值达到了某个阶段的阈值
- 某些剧情在反复对话后才出现 → 这些是需要积累数值或完成前置事件才能解锁的内容

**Tips：**

- 不用刻意"触发"事件，自然地和角色交流就好
- 有些事件有随机性，同一个条件下不一定每次都触发
- 如果你觉得角色变得冷淡了，可能是好感度下降了，多聊聊天就能恢复
- 作者可能会在关键剧情节点让你手动选择（比如表白成功还是失败），这会影响后续剧情走向

## 世界状态

天气、时间、地点等环境信息可能会影响事件的触发。比如某些事件只在下雨天发生，某些约会只在晚上触发。这些由系统自动管理，你不需要手动设置。
