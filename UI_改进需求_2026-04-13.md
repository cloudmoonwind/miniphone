## 实现分析

### 一、全局 CSS 变量

当前状态：index.css 中已有 `:root` 设计令牌（`--fs-*`, `--sp-*`, `--r-*` 等），但各 App 内部 className 仍用 Tailwind hardcode（如 `text-sm`, `p-4`），**未引用**这些变量。用户的问法("是按这个全局设置来的吗")答案是**否**，当前各 App 内部独立，只有桌面图标/Dock 做了统一。

### 二、美化 App 增强

需增加：
1. **桌面透明度** — HomeScreen 的黑色遮罩 `bg-black/10` 改为可变 `opacity`，用 CSS 变量或 prop 传入。状态存 localStorage。
2. **导入字体** — 支持上传 `.ttf`/`.otf`/`.woff`/`.woff2` 字体文件，用 `FileReader` 读成 base64/ArrayBuffer，通过 `FontFace` API 注入到 document.fonts，并更新 `--font-family` CSS 变量（需在 body 样式里引用）。字体信息存 localStorage。

BeautifyApp 自身 UI 也需要用毛玻璃风格重写（脱离 PlaceholderApp 的白底结构）。

### 三、知识库 App（WorldbookApp）改版

#### 3.1 整体视觉
- 背景：`linear-gradient(135deg, #e0c3fc 0%, #8ec5fc 100%)`
- 顶栏、卡片、弹窗：`bg-white/20 backdrop-blur-[16px] border border-white/30`
- 文字适当加 `text-white` / `drop-shadow`

#### 3.2 世界书 Tab 重构
**功能按钮区**（原"新建"按钮旁）：
- 新建 / 导入(JSON) / 导出(JSON) / 复制 / 设置
- 导入：FileReader 读 JSON → POST /api/worldbook/books + /entries
- 导出：GET books+entries → JSON.stringify → download
- 复制：选中书后 duplicate（POST 新书 + 新条目）
- 设置弹窗：
  - 全局扫描深度（默认20，localStorage）
  - 上下文百分比（默认30%，localStorage）

**全局注入面板**（功能区下方）：
- 竖向列表，每行=一本书名
- 点击选中 → 底色高亮 = 启用该书（写回 `book.enabled`）
- 可多选
- 取消世界书书名旁边单独的眼睛图标

#### 3.3 条目 (BookDetail) 折叠展开重构
不再跳转到 EntryForm 页面，改为 **accordion inline**：

**收起态**（单行紧凑）：
```
[条目名] _________ [●策略圆点(可点击切换)] [插入位置badge]
                   [深度/排序/概率 小字]  [复制图标] [删除图标]
```
- 策略圆点：蓝色=常驻, 绿色=关键词；点击直接切换并保存
- 没有指定深度插入时，深度不显示

**展开态**（展开后）：
- 可调高 textarea（content，用 resize 或 drag handle）
- 关键词输入（strategy=keyword 时）
- 插入位置下拉
- 排序、概率
- 高级（递归、分组、定时）折叠块
- 自动保存（onBlur）或显式保存按钮

**移除**：条目左侧眼睛图标（enable/disable）

### 四、实现顺序

1. App.tsx — 增加 desktopOpacity + fontFamily state，透传
2. BeautifyApp.tsx — 重写 UI，加透明度 slider + 字体导入
3. HomeScreen.tsx — 接收并应用 opacity
4. WorldbookApp.tsx — 整体大改（背景 + Tab 按钮 + 设置弹窗 + 全局注入面板 + inline accordion）
5. 后端（可选）— worldbook settings API（全局扫描深度/上下文%），或先用 localStorage
